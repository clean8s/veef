Try closing it with Esc, the close icon or the area behind the modal.

The only thing needed to make a dialog is the HTML inside it. Example: <v-dialog id="myDialog"> This is a <b>dialog</b> </v-dialog>

Opening/closing a dialog

In order to open a dialog you can either use the open="1" HTML attribute or the element.open JavaScript property: document.getElementById('myDialog').open = true; // or <v-dialog open="true" ...

Styling

The v-dialog element uses the original style of the children you pass to it, and then centers the children evenly.
It can also center and stretch buttons for actions, which can be done using the slot slot="actions". A full example with icons, text and actions: <v-dialog id="d1"> <v-icon name="Delete" data-veef="message"></v-icon> <div> <h1>Some title</h1> <h3>Arbitrary HTML. The border on the left happens because the host page has full control over the styling of the dialog (which happens to add a border) while the rest is controlled by veef.</h3> </div> <section slot="actions"> <button primary onclick="d1.open = false">Okay</button> <button onclick="d1.open = false">Meh</button></section> </v-dialog>

Putting your <table> in v-table

To make your table interactive, you need to put your table as a child: <v-table> <table> <tr> <th>Col 1</th> <th>Col 2</th> </tr> <tr> <th>Row 1, 1</th> <th>Row 1, 2</th> </tr> </table> </v-table>

Selection event

You can use JavaScript to detect when a row is selected with the rowselect event. A list of all rows is available in event.detail. <v-table id="myTable"> <table> .. </table> </div> <script> var t = document.getElementById('myTable'); t.addEventListener('rowselect', (e) => { console.log(e.detail); // list of rows }); </script>

<v-icon> A small collection of icons

v-icons contains a small curated list of icons.
The syntax for showing an icon: <v-icon name="Calendar"></v-icon> Note that you must close the tag: </v-icon> because of how Web Component API works.

Changing color

The icons inherit the text color from CSS. If your parent or the v-icon itself have acolor: #ff0000, the icon will be red.

Changing size

The icons can be resized using the CSS width and height properties: <style> v-icon { width: 50px; height: 50px; } </style>

A cool trick

If you ever forget the name of an icon, you can use <v-icon all></v-icon> icon to show all the icons.

Using v-tab is just a matter of adding buttons and content in the right order as children: <v-tabs> <button slot="tab">Tab 1</button> <button slot="tab">Tab 2</button> <div>Content for Tab 1</div> <div>Content for Tab 2</div> </v-tabs> The tab buttons must have slot="tab" while the content doesn't need a slot attribute, but the length of the slotted buttons and the content elements (which don't have to be div) must match.

Web Components and preact

Web Components are a suite of APIs that let you create custom HTML tags. You can read the specification here. The most important parts to know are:
  • The syntax is same as usual tags: <v-icon>, <somelibrary-blah> etc.
  • Some components have children and some don't
  • All components need to have their tags closed (even empty ones), unlike <br/> or <link>
  • An author of Web Component can make them have them appear and behave the way he wants. However, if JavaScript is disabled they will just display the standard HTML content without dynamic events and interactions.

Component children and slotting

A single component can contain children that are placed at different places. For example, you could program a dialog component can have a title part, a content part and action buttons:

<some-dialog> <div slot="title" class="mytitle"> Dialog title </div> <div slot="content"> .. </div> <input slot="button" type="button"> Cancel </input> <some-dialog>
To specify where you want to put a child, you use the slot="" attribute. That attribute is supported on all tags in the browser. By default, all elements in a page have an empty string "" as the default slot.
  • Multiple elements can be put in the same slot.
  • The author of the component decides which slots are supported and how they appear in the final product.

preact syntax

Some of the slots in veef accept preact code. preact is a 3KB library based around the React API for building user interfaces.

It can be used in webpages without transpiling using HTM template literals as described in their tutorials. For example, if you want to render a list of strings in your browser, you can use the following JavaScript:

let strs = ["one", "two", "three"]; let content = strs.map(str => { return h`<li>${str}</li>` }) render(h`<ul>${content}</ul>`, document.body);
Notes about preact and HTM syntax:
  • All tags must be closed, <input> isn't allowed. However, you can use <input />
  • You don't have to escape HTML, `${"<a>"}` will automatically become &lt;a&gt;
  • All sibling tags must have a single parent. If you want to render two elements without a parent, you need the empty fragment syntax: <> <el1></el1> <el2></el2> </>
