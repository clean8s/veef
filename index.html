Web Components is a suite of Web APIs for making interactive components that are just HTML <tags>.

Since they are just usual HTML, they are perfect for non-SPA frameworks like Django, Rails, Laravel.
And you don't need to include build steps or tons of heavy frameworks, stylesheets and scripts.

<v-search> smart fuzzy search

This v-search has 50 Queen songs that it autocompletes. You can type boehman and you'll still find Bohemian!

You can customize the behaviour and you can customize different parts of the element using slot="" and write your own filter and rendering logic.
data object[]
the items you want to autocomplete
searchKey string
each item object must contain this key which will be used for indexing and filtering
searchRender (item: object) => htm
renders a single item

Arbitrary HTML. The border on the left happens because the host page has full control over the styling of the dialog (which happens to add a border) while the rest is controlled by veef.

Try closing it with Esc, the close icon or the area behind the modal.

<v-tree>

A collapsible tree that can be instantiated with a one liner v-json. It's useful for debugging REST APIs or letting users easily explore nested structures. Based around @redux/json-tree
Furthermore, the way nodes are rendered can be customized to make it useful for specific structures:
The most simple way to use it is to supply a JSON string as an attribute: <v-tree data='{"some":"json"}'></v-tree> You can also flip the color scheme by setting the dark attribute: <v-tree dark data='{"some":"json"}'></v-tree> <script> { tree.data = {a: 5}; tree5.renderLabel = (handler) => { } </script>

<v-icon>

The syntax is just <v-icon name="IconName"></v-icon>.
  • The color comes from the text color of the parent.
  • The size comes from the CSS width/height of the element.

If you forget the names of the icons, you can just use <v-icon all> and it will show all the icons like in the demo below:
